    ผมได้ลองทำการสร้าง object หลายๆ object โดยแต่ละอันอาจมี keys ที่เหมือนกันและค่าที่เหมือนกันหรือค่าอาจไม่เหมือนกัน
มาลองกับ function shallow equality เพื่อดูว่าผลจะออกมาเป็นยังไง
หลังจากได้ทำการทดลองผมก็ได้ข้อสรุปว่า 

    shallow equality เป็นการเปรียบเทียบ object หรือ array แบบไม่ลงลึกมาก หรือเช็คแค่ค่าที่อยู่ระดับแรก เช่น ถ้าเป็นตัวเลข
หรือ string ก็เทียบตรงๆ ได้เลย แต่ถ้าข้างในเป็น object หรือ array มันจะเท่ากันก็ต่อเมื่อมันอ้างไปที่ตัวเดียวกันในหน่วยความจำ 
ไม่ได้มานั่งไล่ดูว่า key ข้างในเหมือนกันหมดหรือเปล่า ดังนั้นถ้าเราสร้าง object หรือ array ใหม่ขึ้นมา ถึงจะมีหน้าตาเหมือนกัน
แต่ถ้าไม่ใช่ reference เดียวกัน มันก็จะถือว่าไม่เท่ากันตามหลัก shallow equality
